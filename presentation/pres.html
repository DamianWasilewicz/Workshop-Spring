What is PhoneGap?

PhoneGap is a development framework for mobile applications. Allows users to develop applications using:

  • CSS3
  • HTML
  • JavaScript

History

  • Originally developed by Nitobi
  • Won People's Choice Award at Web 2.0 Expo in 2009
  • Purchased by Adobe
  • PhoneGap is commercial version of Apache Cordova

Why is PhoneGap useful?

  • PhoneGap API allows you to access native OS functionality (camera, location, etc.) using JS
  • You write the app, PhoneGap handles communication with OS
  • Lets you use languages you are familiar with (HTML/CSS, JavaScript) versus facing a learning curve (such as with Objective C or Swift)

Native Application

  • Access to Device
  • Offline-Capable
  • Runs as locally installed application
  • Built for specific devices or operating systems only
  • Built in:
    • Objective-C/Swift (iOS)
    • Java (Android)
    • C# (Windows)

Mobile Application

  • Runs from mobile browser
  • Less access to device
  • No offline capabilities
  • One code for multiple operating systems
  • Built in:
    • HTML5
    • CSS
    • JavaScript

Hybrid: Best of Both Worlds

  • Written like web application (HTML/CSS, JavaScript)
  • Can run as web application
  • One repository
  • Can run as a locally installed app
  • One app for multiple systems
  • Doesn’t require web browser
  • Native API’s (accelerometer, geolocation, etc.)
  • Usually native controls provide navigation/transitions
  • Uses embedded webview (browser bundled inside of a mobile application), which wraps the main content
  • You can make hybrid applications with PhoneGap !!!
    •

Native Plugins

  • PhoneGap does not recreate every native API
  • If you want to access other native functionality, you can write native plugins!

Code Breakdown

Breakdown-config.xml

  • Allows developers to specify metadata about their applications
  • Versions of this file are passively copied into platforms/ subdirectories (eg the config.xml inside browser/)

Breakdown-/hooks

  • Didn’t need any in our project.
  • Hooks are pieces of code that the CLI executes at certain points of your application build process.
  • Named based on when they should execute (after_build, after_run, etc.)
  • Can also store hooks in your config.xml file
  • Typically written in Node.JS
  • Examples of common uses: add plugins, replacing text on the screen, splash screens

Breakdown-/platforms

  • platforms.json file specifies the platforms the app should work on

Breakdown-/www>

  • Images go into the /img subfolder
  • CSS files go into the /css subfolder
  • JS files and Persistence Stores go into the /js subfolder
  • Index.html is the base of the application
